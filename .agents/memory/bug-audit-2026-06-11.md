## Fixed bugs

**Why:** Full deep audit after user reported: infinite loops, token waste, uncalled functions, tool errors, editor bugs, screenshot issues.

### Critical (16 total)
1. `POST /api/history` missing — added to workspace.js; frontend was getting 404 silently
2. `summarizeContext` parsed plain-text as JSON — now uses raw LLM output directly
3. `MAX_STEPS` 100→30, `SWARM_MAX_STEPS` 100→15 — prevents token explosion and loops
4. Swarm runner infinite loop: `toolsUsed===0` had no retry cap — added `MAX_NO_TOOL_RETRIES=3`
5. `messages` array grew unbounded (200+ msgs at 100 steps) — `trimMessages()` keeps last 30
6. `bottomRef` declared but never scrolled — added `useEffect` that calls `scrollIntoView`
7. `stepCount` stale closure in `send()` — replaced with `stepCountRef` for accurate tracking
8. `screenshot.js` dead unreachable else-if — removed
9. Telegram `/clear` called `apiPost("/api/state", {})` (POST → no handler) — replaced with proper DELETE
10. `cli/agent.py` `messages[2:]` skipped first history entry — fixed to `messages[1:]` throughout
11. Swarm synthesis tried `JSON.parse` on plain-text summary — now uses raw text directly
12. `appendLog` had no error handling — wrapped in try/catch
13. `cli/tools.py` screenshot hardcoded Termux path — replaced with `Path(__file__).parent.parent` relative
14. Client-disconnect abort: server agent loop continued after SSE close — added `req.on('close')` + `aborted` flag
15. History from previous session re-appended on every run — `cappedHistory.slice(-MAX_HISTORY_MESSAGES)` before merge
16. `MAX_HISTORY_MESSAGES=30` constant added to shared/constants.js

**How to apply:** Check these files for any future regressions: `server/routes/agent.js`, `server/routes/workspace.js`, `swarm/runner.js`, `src/hooks/useAgent.js`, `cli/agent.py`, `shared/constants.js`
