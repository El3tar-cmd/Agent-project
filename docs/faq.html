Frequently Asked Questions

Answers to common questions about setup, security, and usage of the Coding Agent.

Why do I get a 404 error when clicking some navigation links?

This documentation was initially created with a template navigation that included placeholder links for architecture.html and faq.html. These pages have now been created to complete the documentation suite. If you're still seeing 404 errors, make sure you've refreshed your browser after the files were created.

Is it safe to run this agent on my machine?

The agent is designed with security in mind:

  • All file operations are restricted to the project directory
  • Shell commands are executed with safety checks
  • No sensitive data is stored or transmitted
However, it's recommended to run the agent in a sandboxed environment (Docker container or VM) for maximum security.

Which LLM models work best with this agent?

The agent is optimized for code generation and analysis. Recommended models:

  • qwen2.5-coder:7b (default, excellent for coding)
  • llama3:8b-code (good alternative)
  • deepseek-coder:6.7b (specialized for code)
You can switch models using the /model command in CLI mode.

How do I add a new tool to the agent?

To add a new tool:

  1. Create or define the tool function under the server/tools/ directory
  2. Register the tool in the server/tools/index.js dispatcher map
  3. Update the documentation in docs/tools.html
Make sure to implement proper input validation and error handling for security.

How often should I update the agent?

The agent is designed to be stable, but you should update when:

  • New security patches are available
  • You need to support new tools or features
  • There are performance improvements
Check the project's GitHub repository for the latest updates.
