Tool Reference

The agent interacts with your system through a set of predefined tools. These tools allow it to read, write, execute, and explore your codebase autonomously.

Tool Name Description Use Case
read_file Reads the full content of a specific file. Analyzing existing code or reading config files.
write_file Creates a new file or overwrites an existing one. Implementing new features or creating scripts.
replace_text Performs a targeted find-and-replace in a file. Applying bug fixes or small logic changes.
run_command Executes a shell command and returns the output. Running tests, installing packages, or checking logs.
list_files Lists all files and folders in a directory. Exploring the project structure.
search_in_files Recursively searches for a pattern across files. Finding where a function or variable is used.
create_dir Creates a new directory. Organizing new modules or assets.
delete_file Deletes a file or directory. Cleaning up temporary files or removing old code.
http_get Fetches content from a URL. Reading API documentation or external resources.
python_eval Executes a snippet of Python code. Performing complex calculations or data processing.
git_status Shows the current git status. Checking for uncommitted changes.
git_diff Shows the diff of changes. Reviewing what the agent has modified.
grep Searches for a pattern using grep. Fast text searching in large files.

Security Note

Because the agent has the ability to execute shell commands (run_command) and modify files, it is highly recommended to run the agent in a sandboxed environment (such as a Docker container or a dedicated VM). Never run the agent with root/administrator privileges on a production machine.
